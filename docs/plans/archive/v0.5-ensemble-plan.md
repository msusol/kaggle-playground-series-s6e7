# v0.5 - Ensemble Plan (Rung 4)

Goal: check whether blending diverse model families beats the current best single
model (v0.3 CatBoost, OOF ~0.949, LB ~0.949). Per `implementation-plan.md`'s Rung 4:
"Ensemble LightGBM + CatBoost + a regularized linear/NN baseline."

Given the Rung 3 finding (`docs/investigate/2026-07-03-kaggle-discussion-findings.md`)
that this dataset likely has a ~0.95 synthesis-noise ceiling, and that LightGBM and
CatBoost already converge to a similar range (suggesting correlated errors, since
both are learning much the same near-deterministic rule + noise), the ensemble's
realistic upside is probably small. Still worth trying — cheap relative to the
information gained — but budget effort accordingly and report honestly either way.

## Approach

Reproduce each member model's exact validated config (no re-tuning — Rung 4 is about
combining existing best efforts, not further tuning individuals) capturing OOF *and*
test probabilities this time (v0.1 and v0.3's original harnesses only saved hard
labels for OOF):

1. **LightGBM** — v0.1's exact config (base features, `class_weight='balanced'`,
   `n_estimators=2000, learning_rate=0.05`). OOF 0.9389 / LB 0.94051.
2. **CatBoost** — v0.3 Variant 1's exact config (base features,
   `auto_class_weights='Balanced'`, custom balanced-accuracy eval metric). OOF
   0.9493 / LB 0.94885.
3. **New: regularized multinomial logistic regression** — genuine architectural
   diversity from the two tree ensembles. Needs different preprocessing (one-hot
   categoricals with an explicit `"missing"` level, median-impute numerics + a
   missing-indicator per column since linear models can't handle NaN natively,
   standardize). Expected to score notably lower alone, but may still help an
   ensemble if its errors are sufficiently uncorrelated with the tree models'.

Blend the OOF probability matrices with a weighted average, search blend weights via
grid search + **nested validation** (fit weights on 4/5 folds, evaluate on the
held-out 5th, cycle across folds) — same honest-estimate discipline as Rung 3, since
naively fitting blend weights on the same OOF set used to report the score would be
optimistic.

## Steps

1. Reproduce LightGBM v0.1 and CatBoost v0.3 Variant 1, capturing OOF + test
   probabilities this time.
2. Build and fit the logistic regression baseline with proper preprocessing.
3. Report each model's solo OOF balanced accuracy as a sanity check against known
   results before trusting the probability matrices.
4. Grid search blend weights (3-way, e.g. a simplex grid) to maximize OOF balanced
   accuracy.
5. Nested-validate the best blend weights for an honest improvement estimate.
6. Try pairwise blends too (LightGBM+CatBoost, CatBoost+Linear, LightGBM+Linear) for
   comparison — worth knowing if a subset does better than the full 3-way blend.
7. Write a candidate `submission.csv` only if nested validation confirms a real
   improvement over the current best (~0.9493 OOF / ~0.94885 LB). Do not auto-submit.

## Acceptance

- Each reproduced model's OOF matches its known validated result before trusting
  anything downstream (same reproducibility-check discipline as v0.4).
- The reported improvement (if any) comes from nested validation, not the
  same-data blend-weight fit.
- Findings recorded in `docs/plans/leaderboard.md`, `docs/investigate/notebook-runs.md`,
  and `docs/plans/implementation-plan.md` regardless of outcome.
