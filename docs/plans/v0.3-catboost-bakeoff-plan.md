# v0.3 - CatBoost Bake-off Plan

Goal: test whether CatBoost beats LightGBM v0.1 (OOF 0.9389, LB 0.94051, still the
best model after v0.2's negative feature-engineering result). Directly acts on two
Rung 2 lessons:

1. LightGBM's early stopping tracked `multi_logloss`, not balanced accuracy, and more
   training rounds under that proxy metric made CV *worse*. CatBoost supports custom
   Python eval metrics — implement a real `balanced_accuracy_score`-based metric so
   early stopping tracks the actual competition metric this time.
2. v0.2's manually engineered features (crosses + target encoding) added variance
   that net-hurt LightGBM despite one cross becoming the top feature by importance.
   CatBoost's ordered boosting and native categorical handling may absorb that same
   signal with less overfitting risk.

## Steps

1. Same 5-fold stratified CV split (identical `random_state=42`) as v0.1/v0.2, for a
   rigorous apples-to-apples comparison.
2. **Variant 1**: CatBoost on v0.1's exact 13 base features (raw categoricals as
   strings with `NaN` -> `"missing"`, no need for shared pandas Categorical codes —
   CatBoost hashes category strings internally). `auto_class_weights='Balanced'`,
   custom balanced-accuracy eval metric, early stopping.
3. **Variant 2**: CatBoost on v0.2's full 35-feature engineered set (missingness
   indicators, categorical crosses, OOF target encoding) — same custom metric/early
   stopping — to see whether CatBoost handles that larger feature set without v0.2's
   regression.
4. Compare OOF balanced accuracy, per-class recall, and feature importance against
   v0.1 (0.9389), v0.2 Section A (0.9290), v0.2 Section D (0.9255).
5. Write a candidate `submission.csv` from whichever model is actually best across
   all runs so far (do not blindly prefer the newest experiment).
6. Do not auto-submit to Kaggle — ask first.

## Acceptance

- CV harness reuses the identical fold split for a fair comparison.
- Both variants smoke-tested on a data sample before committing to the full run
  (the v0.2 experience: a real bug wasted ~37 minutes of compute on the first attempt).
- Findings recorded in `docs/plans/leaderboard.md`, `docs/investigate/notebook-runs.md`,
  and `docs/plans/implementation-plan.md` (Rung 2 CatBoost follow-up).
