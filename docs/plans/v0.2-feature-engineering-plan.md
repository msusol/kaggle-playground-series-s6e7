# v0.2 - Feature Engineering Plan (Rung 2)

Goal: check whether v0.1 (`notebooks/v0.1-baseline.ipynb`, OOF balanced accuracy
0.9389, LB 0.94051) was under-trained, then see whether engineered features move the
needle beyond what LightGBM's native categorical splits already extract from
`stress_level` / `physical_activity_level` — the two dominant signals from the EDA.

## Steps

1. **Training-budget ablation** (same features as v0.1): none of v0.1's 5 folds
   early-stopped at `n_estimators=2000`. Re-run with a higher round budget / lower
   learning rate on the *same* feature set to isolate how much of any future gain
   comes from more training vs. from new features.
2. **Root-cause the `sleep_duration` surprise**: v0.1's feature importance ranked
   `sleep_duration` (numeric) #2, which the v0.1-eda univariate histograms didn't
   predict. Binned/quantile view of `sleep_duration` vs. `health_condition`, and a 2D
   cut against `stress_level` / `physical_activity_level`, before deciding what to
   engineer.
3. **Engineered features**:
   - Missingness indicators for numeric columns (categorical NaNs already have an
     explicit `"missing"` level from v0.1, so an indicator there is redundant).
   - Categorical x categorical interactions: `stress_level` x
     `physical_activity_level` (the two dominant signals), `sleep_quality` x
     `smoking_alcohol` (the two secondary signals).
   - Numeric x categorical interactions informed by step 2's findings (e.g. binned
     `sleep_duration` x `stress_level`).
   - Smoothed out-of-fold multiclass target encoding for the base categoricals and
     the new interaction crosses (fit within each training fold only, to avoid
     leakage; test-set encoding averaged across the 5 folds).
4. **Retrain**: same 5-fold stratified CV harness, tuned training budget from step 1,
   full engineered feature set. Compare OOF balanced accuracy against v0.1.
5. **Feature importance**: check whether engineered features rank meaningfully, or
   whether LightGBM's native categorical splits already captured the same signal
   (expected for low-cardinality categoricals — worth confirming rather than assuming).
6. Write a candidate `submission.csv`; do not submit automatically (submission
   consumes a daily quota and is an explicit user decision).

## Acceptance

- OOF balanced accuracy improves over v0.1's 0.9389, or the notebook records *why* it
  didn't (e.g. native categorical splits already saturated the signal).
- CV harness stays 5-fold stratified, comparable to v0.1's.
- Findings recorded in `docs/plans/leaderboard.md`, `docs/investigate/notebook-runs.md`,
  and `docs/plans/implementation-plan.md` (Rung 2 status).
